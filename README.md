# Сопоставление НМ

> 📄 **Обсуждение и подробная инструкция:** статья на **Инфостарте** — *ссылка будет добавлена после публикации.*

Внешняя обработка для **1С:Предприятие** (вид: дополнительная обработка): семантическое сопоставление строк сметы с позициями справочника **Номенклатура** через эмбеддинги и векторный поиск.

---

## 🔗 Связанные репозитории

| Репозиторий | Назначение |
|-------------|------------|
| 📦 [**Выгрузить НМ**]([https://github.com/YOUR_ORG/vygruzka-nm](https://github.com/egordoronchenko/vygruzka-nm)) | Индексация справочника **Номенклатура** в ChromaDB |
| 🔍 **Сопоставить НМ** | Этот репозиторий: поиск по индексу, сопоставление сметных позиций |



---

## 🔄 Порядок в контуре НМ

**Сначала** выполняется индексация справочника в Chroma внешней обработкой **«Выгрузить НМ»** — иначе в коллекции не будет векторов для поиска.

👉 **Репозиторий выгрузки:** [Выгрузить НМ на GitHub](https://github.com/egordoronchenko/vygruzka-nm).

**Затем** запускается эта обработка — она только **ищет** по уже заполненной коллекции и **не заменяет** выгрузку.

---

## 🎯 Назначение

1. Загрузить наименования сметных позиций из **Excel** (`.xlsx` / `.xls`) и ввести их в табличной части **«Сметные позиции»**.
2. Выполнить команду **«Сопоставить НМ»**: для каждой строки строится запрос к коллекции в **Chroma**, вектор запроса получается в **LM Studio** (модель эмбеддингов).
3. В дереве на форме отображаются кандидаты из номенклатуры с оценкой сходства (по умолчанию — совпадения с **сходством ≥ 70%**).

---

## 📋 Что нужно заранее

- ✅ Развёрнутые и доступные с информационной базы **LM Studio** (REST API эмбеддингов) и **Chroma** (HTTP API v2).
- ✅ В Chroma — коллекция с эмбеддингами номенклатуры и метаданными с `guid` позиций 1С (типичный сценарий — предварительная выгрузка каталога той же инфраструктурой, что и обработка **«Выгрузить НМ»**).

---

## 🚀 Установка

1. Соберите внешнюю обработку (`.epf`) из выгрузки конфигурации или подключите каталог как проект в EDT.
2. В базе: **Администрирование → Печатные формы, отчеты и обработки → Дополнительные отчеты и обработки** — загрузите файл обработки и включите использование.

---

## ⚙️ Настройка подключений

Параметры LM Studio и Chroma (хост, порт, имя коллекции, модель эмбеддингов, размер батча и т.д.) задаются в коде модуля формы в функции **`НастройкиВыгрузкиНМ()`** — при переносе на другой стенд отредактируйте значения под свою среду.

Порог сходства и число кандидатов на строку задаются в процедуре **`СопоставитьНМНаСервере()`**.

**Модуль формы:** `СопоставлениеНМ/Forms/Форма/Ext/Form/Module.bsl`

---

## ⚠️ Ограничения

- Обработка рассчитана на работу в связке с **той же схемой метаданных Chroma**, что и при индексации номенклатуры (идентификация по **GUID** в метаданных записи).
- Порог **0,7** и число кандидатов **20** на строку заданы в процедуре **`СопоставитьНМНаСервере()`** — при необходимости меняются в коде.
